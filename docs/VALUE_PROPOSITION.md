# 项目价值

## 面向场景

- 分布式缓存与对象存储；
- 消息分区、任务队列和租户路由；
- 搜索索引、时序数据和边缘节点；
- 需要离线评审拓扑变更的控制面工具。

## 真正解决的问题

稳定哈希只能给出节点分数，不能回答以下运维问题：

- 三副本是否跨区域、跨机架；
- 新增节点会移动多少键；
- 哪些数据应先回填、何时切换主节点；
- 如何限制单批迁移量，避免热点节点被压垮；
- 拓扑异常时哪些键必须停止迁移。

MoonShardKit 将这些问题建模为确定性算法。相同快照和参数在不同后端产生
相同结果，因此计划可以在上线前审查、在 CI 中回归，并被控制面重复生成。

## 差异化

项目的核心差异不是再实现一个哈希函数，而是把放置决策推进到“可执行但不
绑定执行环境”的安全迁移工作流：

1. 先创建目标副本；
2. 从现有可读副本回填；
3. 校验目标副本；
4. 必要时切换主节点；
5. 最后删除旧副本。

迁移动作按全局并发和单节点压力预算分波次。没有可读源的数据不会生成伪安全
计划，而会进入阻塞清单。

## 后端中立

核心不读取时钟、不使用随机数、不连接网络和数据库，也不依赖线程。它适合在
浏览器 Wasm、Node.js、边缘运行时和 Native 控制面中复用。
